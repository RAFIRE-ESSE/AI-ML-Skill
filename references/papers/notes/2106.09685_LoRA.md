## Key Innovation

Freezes pre-trained model weights $W_0 \in \mathbb{R}^{d \times k}$ and injects trainable rank decomposition matrices into each layer of the Transformer architecture. This drastically reduces the number of trainable parameters for downstream tasks by up to 10,000x and GPU memory requirement by 3x without inference latency penalty.

---

## Core Formulations

$$h = W_0 x + \Delta W x = W_0 x + \frac{\alpha}{r} B A x$$

- $W_0 \in \mathbb{R}^{d \times k}$: Frozen pretrained weight matrix.
- $B \in \mathbb{R}^{d \times r}$ and $A \in \mathbb{R}^{r \times k}$: Trainable low-rank matrices, where rank $r \ll \min(d, k)$.
- $\alpha$: Constant scaling factor ($\frac{\alpha}{r}$).
- **Initialization**: $A \sim \mathcal{N}(0, \sigma^2)$ and $B = 0$, ensuring $\Delta W = 0$ at the start of training.

---

## Advantages

1. **No Inference Latency**: At deployment, $\Delta W = B A$ can be explicitly folded into $W_0$ ($W_{eff} = W_0 + \Delta W$), requiring zero extra computation.
2. **Modular Parameter Storage**: Switching between downstream tasks requires saving only small adapter matrices (megabytes instead of gigabytes).

import torch
import torch.nn as nn

class LoRALinear(nn.Module):
    def __init__(self, in_features, out_features, rank=8, alpha=16):
        super().__init__()
        self.linear = nn.Linear(in_features, out_features, bias=False)
        self.linear.weight.requires_grad = False  # Freeze main weight
        
        self.rank = rank
        self.scaling = alpha / rank
        self.lora_A = nn.Parameter(torch.zeros(rank, in_features))
